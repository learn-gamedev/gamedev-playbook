# Игровой цикл (Game Loop)

**Игровой цикл** — центральный паттерн архитектуры игры: бесконечный цикл, который на каждом кадре обрабатывает ввод, обновляет состояние мира и отрисовывает сцену.

```
while running:
    process_input()   # читаем события
    update()          # обновляем состояние
    render()          # рисуем кадр
```

---

## Три фазы цикла

### 1. Process Input (Обработка ввода)
Считываем все события, накопившиеся с прошлого кадра: нажатия клавиш, движение мыши, закрытие окна.

```python
for event in pygame.event.get():
    if event.type == pygame.QUIT:
        running = False
    if event.type == pygame.KEYDOWN:
        if event.key == pygame.K_SPACE:
            player.jump()
```

### 2. Update (Обновление состояния)
Двигаем объекты, проверяем коллизии, обновляем ИИ, изменяем счёт.  
Все изменения — только здесь, не в render.

```python
def update(dt):
    player.update(dt)
    enemies.update(dt)
    check_collisions()
```

### 3. Render (Отрисовка)
Очищаем экран, рисуем все объекты в нужном порядке (слоями), обновляем дисплей.

```python
def render():
    screen.fill(BLACK)
    background_group.draw(screen)
    sprite_group.draw(screen)
    ui_group.draw(screen)
    pygame.display.flip()
```

## Управление временем: Delta Time

Без учёта времени скорость объектов зависит от FPS — на быстрой машине игра идёт быстрее.

**Дельта-время (dt)** — время, прошедшее с прошлого кадра (в секундах). Умножая скорость на `dt`, получаем движение, независимое от FPS.

```python
clock = pygame.time.Clock()
FPS = 60

while running:
    dt = clock.tick(FPS) / 1000  # миллисекунды → секунды

    process_input()
    update(dt)   # передаём dt в update
    render()

# В update:
self.x += self.speed * dt  # пикселей в секунду, не в кадр
```

## Типы игровых циклов

### Variable Timestep (переменный шаг)
Простейший вариант: `dt` зависит от реального времени рендера.  
**Плюс:** прост в реализации.  
**Минус:** физика нестабильна при резких просадках FPS.

```python
dt = clock.tick() / 1000
update(dt)
```

### Fixed Timestep (фиксированный шаг)
Физика обновляется с фиксированным шагом, рендер — как можно чаще.  
Используется в играх с точной физикой или сетевым мультиплеером.

```python
FIXED_DT = 1 / 60
accumulator = 0.0

while running:
    frame_time = clock.tick() / 1000
    accumulator += frame_time

    process_input()
    while accumulator >= FIXED_DT:
        update(FIXED_DT)
        accumulator -= FIXED_DT
    render()
```

### Semi-Fixed (полуфиксированный)
Компромисс: шаг фиксированный, но при большой задержке шаги объединяются.  
Применяется в большинстве коммерческих движков (Unity: `FixedUpdate` + `Update`).

## Управление состояниями (State Machine)

Реальная игра переключается между экранами. Простейший способ — флаг состояния:

```python
state = "menu"  # "menu" | "playing" | "paused" | "game_over"

while running:
    process_input()

    if state == "menu":
        update_menu()
        render_menu()
    elif state == "playing":
        update_game(dt)
        render_game()
    elif state == "game_over":
        render_game_over()
```

Для сложных игр — словарь состояний или паттерн [[ООП|Strategy/State]].

## Полный шаблон на Qt (PyQt6 / PySide6)

В Qt нет явного `while`-цикла: его заменяет `QTimer`, который вызывает игровой тик с заданным интервалом. Рендер происходит в `paintEvent`, ввод — в `keyPressEvent` / `keyReleaseEvent`.

```python
import sys
from PyQt6.QtWidgets import QApplication, QMainWindow
from PyQt6.QtCore import QTimer, QElapsedTimer, Qt
from PyQt6.QtGui import QPainter, QColor


class GameWindow(QMainWindow):
    FPS = 60

    def __init__(self):
        super().__init__()
        self.setWindowTitle("My Game")
        self.setFixedSize(800, 600)

        self._elapsed = QElapsedTimer()
        self._elapsed.start()
        self._last_ms = 0

        self.keys = set()  # множество зажатых клавиш

        self._timer = QTimer(self)
        self._timer.timeout.connect(self._tick)
        self._timer.start(1000 // self.FPS)  # ~16 мс

    # ── 1. Input ───────────────────────────────────────
    def keyPressEvent(self, event):
        self.keys.add(event.key())

    def keyReleaseEvent(self, event):
        self.keys.discard(event.key())
        if event.key() == Qt.Key.Key_Escape:
            self.close()

    # ── 2. Update ──────────────────────────────────────
    def _tick(self):
        now = self._elapsed.elapsed()
        dt = (now - self._last_ms) / 1000.0  # секунды
        self._last_ms = now

        self._update(dt)
        self.update()  # запрашивает перерисовку → paintEvent

    def _update(self, dt):
        # обновление объектов
        pass

    # ── 3. Render ──────────────────────────────────────
    def paintEvent(self, event):
        painter = QPainter(self)
        painter.fillRect(self.rect(), QColor(0, 0, 0))  # очистка
        # ... рисуем объекты через painter ...
        painter.end()


if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = GameWindow()
    window.show()
    sys.exit(app.exec())
```

### Отличия от pygame

| | pygame | Qt (PyQt6) |
|---|---|---|
| Цикл | `while running` | `QTimer.timeout` |
| Время | `clock.tick(FPS)` | `QElapsedTimer` |
| Ввод | `pygame.event.get()` | `keyPressEvent` / `keyReleaseEvent` |
| Рендер | `screen.fill()` + `display.flip()` | `paintEvent` + `QPainter` |
| Запуск | `pygame.init()` | `QApplication` + `app.exec()` |

> PySide6 — прямой аналог PyQt6: достаточно заменить `from PyQt6` на `from PySide6`.
